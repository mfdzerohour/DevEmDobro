O React Query lida com o gerenciamento de estado de consultas automaticamente, reduzindo a necessidade dos desenvolvedores escreverem e manterem lógicas complexas de gerenciamento de estado. Ele fornece recursos integrados de tratamento de erros, permitindo que os desenvolvedores lidem com erros de API normalmente.

Principais recursos:

  • Queries => Recuperação de dados
  • Mutations => Atualização e criação de dados
  • Invalidation: Invalidação de cache

Caracteristicas do React Query

  • Caching
  • Refetching ao focar a janela
  • Capacidade de diminuir o número de tentativas de requisições
  • Prefetching
  • Atualizações otimistas

Principais benefíciosy

  • Gerenciar o estado da requisição
  • Colocar dados recebidos de forma mais facil
  • Atuailização constante com os dados do servidor
  • Abstrações para evitar a repetição de lógica

Hook UseQuery

  • uSeQuery para realizar consultas
  • Sintaxe básica: useQuery('chave', funçãoAssíncrona)

Principais propriedades do useQuery

  • data: Os dados resultantes da consulta, se ela for bem-sucedida
  • isLoading: Indica se a consulta esta atualmente em andamento
  • isError: Indica se ocorreu um erro durante a execução da consulta
  • error: Fornece informações detalhadas sobre o erro, se houver.
  • refetch: Função que permite refazer manualmente a consulta
  • isFetching: Indica se a consulta está atualmente sendo "rebuscada" em segundo plano para atualizar o dados em cache.

Hook useMutation

  • useMutation para realizar operações de mutação, como atualizar ou criar dados.
  • Sintaxe básica: useMutation (funçãoAssíncrona)

Principais propriedades do useMutation

  • onSuccess, onError: Tratamento de sucesso ou erro
  • onSettled: Executa após a mutação independente do resultado
